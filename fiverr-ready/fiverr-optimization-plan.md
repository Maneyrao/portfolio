# Fiverr Optimization Plan - Thiago Maneyro

## Objetivo

Convertir el perfil de Fiverr en una oferta clara de desarrollo web, e-commerce, sistemas internos y automatizaciones con IA para negocios que necesitan vender mejor, ordenar procesos y verse mas profesionales.

## Informacion que vamos a usar

- Nombre y marca: Thiago Maneyro / TM.
- Posicionamiento: desarrollador de software orientado a negocios, conversion, procesos y presencia digital.
- Prueba real:
  - Club Amsterdam: sistema de reservas y panel admin.
  - IG Detail: e-commerce.
  - Sebastian Esquivel Propiedades: web institucional inmobiliaria premium.
  - De Fazio Propiedades: web inmobiliaria moderna con integracion Xintel.
- Stack tecnico:
  - Next.js, React, Tailwind CSS, FastAPI, PostgreSQL, Supabase, Railway, Vercel e integraciones con IA.
- Valores:
  - Compromiso.
  - Transparencia.
  - Resultados.
- Restricciones de Fiverr:
  - No usar WhatsApp, email ni telefono en gigs, imagenes o descripciones.
  - Mantener comunicacion y contratacion dentro de Fiverr.
  - Imagenes recomendadas: 1280 x 769 px.
  - Portfolio projects con descripcion minima de 120 caracteres y al menos un archivo visual.

## ICP principal

1. Small business owner
   - Necesita web, tienda, reservas o sistema simple.
   - Le importa vender, ahorrar tiempo y verse profesional.
   - No quiere una explicacion tecnica eterna; quiere saber que problema se resuelve.

2. Real estate professional
   - Necesita mostrar propiedades y transmitir confianza.
   - Valora estetica, claridad, velocidad y presencia premium.

3. Service business
   - Barberias, estudios, turnos, consultorios, academias, marcas locales.
   - Necesita agenda, dashboard, automatizaciones o flujo de consultas.

4. Product brand / e-commerce starter
   - Necesita catalogo, tienda, estructura de productos, combos y contacto.
   - Busca una primera presencia digital fuerte.

## Orden de ejecucion

1. Completar perfil publico.
2. Subir proyectos al portfolio interno de Fiverr.
3. Crear 4 gigs iniciales.
4. Usar thumbnails diferentes para cada gig.
5. Preparar FAQs y requisitos de compra para reducir preguntas repetidas.
6. Pedir primeras reviews dentro de Fiverr cuando entren trabajos.

## Gigs iniciales

1. Modern business website
   - Objetivo: webs institucionales, portfolios y sitios profesionales.
   - Caso que lo prueba: Sebastian Esquivel / De Fazio.

2. Landing page
   - Objetivo: campañas, validacion, captacion de leads, consultas.
   - Caso que lo prueba: De Fazio / portfolio propio.

3. E-commerce
   - Objetivo: tienda, catalogo, productos, estructura lista para pagos.
   - Caso que lo prueba: IG Detail.

4. Booking system / dashboard / AI automation
   - Objetivo: reservas, paneles internos, workflows y automatizacion.
   - Caso que lo prueba: Club Amsterdam.

## Regla de copy

No vender tecnologia primero. Vender resultado:

- Mas consultas.
- Mejor confianza.
- Menos trabajo manual.
- Mejor experiencia de usuario.
- Una herramienta real para operar o vender.

La tecnologia se usa como respaldo, no como headline principal.
